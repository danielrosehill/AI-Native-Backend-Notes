An open source notebook of questions and answers about AI-native backend development. Questions are written by Daniel; answers are researched and generated by Claude.

# Directory Structure

```
questions/to-run/   # Pending questions (not yet answered)
questions/run/      # Answered questions (archive)
answers/            # Answer files, optionally grouped into topic subfolders
answers/INDEX.md    # Auto-generated index of all answers
exports/            # Typst templates and PDF exports
```

# Workflow

## Adding Questions

Daniel provides questions individually or in batches (sometimes via dictation). Save each question as a separate markdown file in `questions/to-run/` with a descriptive kebab-case filename.

You should edit the question lightly for clarity while preserving its core detail. 

## Answering Questions

For each file in `questions/to-run/`:

1. Read the question
2. Write a thorough markdown answer in `answers/` (descriptive kebab-case filename, e.g. `best-api-for-weather-data.md`)
3. Move the question file from `questions/to-run/` to `questions/run/`

## Slash Commands

- `/add-questions <text>` - Parse and save one or more questions to `questions/to-run/`
- `/run-prompts` - Answer all pending questions in `questions/to-run/`
- `/organise` - Group answers into topic subfolders and update the index
- `/export` - Generate a timestamped PDF export via Typst

## Subagents

- `researcher` - Researches and answers a single question (uses web search)
- `batch-runner` - Processes all pending questions in parallel via researcher agents

# Maintenance

Run periodically (or use `/organise` and `/export`):

1. Group answers into topic subfolders under `answers/`
2. Update `answers/INDEX.md`
3. Generate a Typst PDF export to `exports/export-DDMMYYYY.pdf` with:
   - Summarised questions followed by full answers
   - Page numbers in footer
   - Generation date in dd/mm/yyyy format
